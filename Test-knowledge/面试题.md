# 在测试人员同开发人员的沟通过程中，如何提高沟通的效率和改善沟通的效果？
+ 目的：在沟通之前要首先将问题的弄得非常清楚，确定自己要对于该问题沟通的方面和要点在哪里，明白沟通的目标是什么。
+ 态度：真诚，不掺杂主观色彩.有团队意识。 
+ 方式：面对面沟通，减少误解出现的可能性。
+ 运用一些管理工具，如testdirector
+ 添加剂：一些小bug可以不经过bug管理系统而直接向开发提出

# 测试与开发的关系如何协调？
我觉得，首先是心态方面，任何一段关系都是建立在平等真诚之上。以一颗平和而且不掺杂有色感情的心去沟通去交流。而且
接着是工作方面。团队意识必不可少。。。。在与开发沟通之前自己先弄清楚沟通的点想要的沟通目标等等进行深思之后再去交流
接着是一些技巧方面，不乱提bug，尽量多的协助开发找到问题根源应该会对二者的关系起到很好的调节作用，或者把一些很小的bug私下提给开发而不经过缺陷管理系统也是一个很好的方法。



# 一个测试工程师应具备那些素质和技能？
+ 技术方面：掌握基本的软件测试基础理论。明白测试是什么，为什么，和怎么做。需要能熟练阅读需求规格说明书等文档，并设计出优秀的测试用例。当然也要了解一点软件工程，软件开发的相关知识。
+ 非技术方面（个人的素养）: 个人的沟通能力（学会描述问题，倾听问题，及时沟通，，并掌握一定的沟通技巧），团队意识，**耐心，细致**（测试是一个繁琐且不断重复的工作），怀疑精神，主观能动能力（主动查资料主动找人沟通）等。

# 怎么自学测试的？
当初决定好去从事测试这个职业是在研二。首先，我是先去图书馆借了几本书。通过阅读这些书籍，再加上去知乎、简书上的了解，我大概明白了这是一个什么样的职业，要涉及什么样的工作内容。也发现了这个工作的意义。正式准备的时候我是首先把要学的知识进行了分块。除了最重要的软件测试基础理论，还包括测试工具和一些辅助工具的使用，计算机基础方面、语言学习方面等等。对于工具方面，我都做到了从最初的下载到最后的实操，争取弄懂每一步操作。当然对于数据库，git这些的工具，我觉得仅仅是可视化操作界面的学习，应该是不够的，所以一直也是在用命令行。我在github上面建了一个项目，用来保存自己的笔记。每次修改后都是按照正常的提交流程在操作，这也算是平时的实操把。现在想想整个测试过程的学习，真的是是边摸索边前进。遇到问题，就去查各种技术文档，当然也请教了不少次前辈。



# 怎么看待目前比较火的短视频？
抓住了用户什么痛点，运营上的优势，爆发点在哪？未来发展空间？
1. 从行业崛起上来说，它满足了用户摄取信息快。短。新的痛点。
2. 从商业模式上来说，短视频通过短且有质量的视频赢得了快速扩散以及点击率。
3. 从运营模式上来说，刺激用户自发创造内容，并且给予奖励和处罚。
4. 从趋势上来说：短视频是信息的最便捷的方式，但要对内容加以把控，不可为盈利不择手段，随着越来越多的同行业崛起，需要在运营方面和内容方面进行创新和整顿。

# 果园
项目经历这一块最开始的时候是先测了一个简单的加购送优惠券的页面。后来是一个果园项目。是我的一个好朋友帮我搭建的一套前后端系统。它从mysql数据库，到node的服务端，再到vue的页面框架，每一个都是现在比较流行的技术。我就是在这一套果园上开始的第一个项目测试。
果园内部涵盖了用户登录注册，不同种类小树的不同生命周期，道具的获得使用以及一个水滴雨小游戏。它内部的业务我觉得还算是比较复杂的。我按照正常的测试流程走了一遍，从了解项目需求，到画思维导图，设计测试用例，去测试每一个功能点，提bug,验证bug,到关闭bug。通过这个项目，我完整的实践了一个项目的测试过程，让我对测试这个职业认识的更加深刻。
