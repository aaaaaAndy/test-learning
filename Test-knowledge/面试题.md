# 具体的项目介绍
1. 加购送优惠券：（页面检查，逻辑检查，按钮检查，兼容性，多语言）--提高产品加购率和用户活跃度
这个活动的实现逻辑比较简单的，就是在活动区域内，只要购物车里面的加购数量满15件就可以参与抽奖，赢得优惠券。但其中的测试点除了页面背景和规则陈列外，抽奖条件需要进行详细测试。比如：所在区域，加购商品的数量，符合条件商品的种类，参与的次数等。每种造成抽奖失败的情况触发后都会有对应的提示弹框和不同的按钮设置。那抽奖成功后，获得的优惠券也是一个比较重要的测试点。比如检查获得的优惠券种类与id是否一致、检查账户里面是否真正的增加了对应的优惠券、优惠券面额是否一致等。另外除了功能测试，页面测试，还进行了语言转换测试和兼容性测试
2. 果园项目
### 项目简介：
果园进去先是一个登录界面，然后进入到主页。主页的中心是一颗小树的界面展示，它会根据不同的果树种类和不同的生命周期显示对应的图片。（种类：椰子树和苹果树，生命周期：幼苗-小树-大树-结果）右下方是一个水壶，点击时可以触发浇水效果。然后小树会经历不同的生命周期，每个生命周期需要的水量不同。这是果园的主流程。页面下方是四个按钮，分别是签到、任务、背包，成就。（签到界面是每日首次浇水后可以签到领取奖励，任务界面有两种，一个是好友邀请，触发后会生成邀请链接。一个是水滴雨小游戏。完成后可收取水滴奖励。成就界面就是每种成一颗小树可点亮相应勋章，背包界面里面有太阳和云朵道具，太阳点击可切换表情，云朵比较强大，它是左右漂浮状态，但每隔六个小时可下雨浇水，浇水时云朵固定在中间，每次浇水量是水壶的3倍）
### 记忆深刻的测试点
1. 按照正常的测试流程走了一遍，从了解项目需求，到画思维导图，设计测试用例，去测试每一个功能点，提bug,验证bug,到关闭bug。通过这个项目，我完整的实践了一个项目的测试过程，让我对测试这个职业认识的更加深刻。
2. 种树阶段，所选择的果树种类是否与页面图片相符。成长阶段，小树的每一个生命周期所需的水量是否与逻辑匹配，跨等级时，界面展示和水量逻辑是否正确。
3. 小技巧： + 在测试跨等级时，会去修改数据库water的值，使它接近跨级所需水量再去点击查看效果
           + 测试果园白天黑夜背景切换时，也会去改变手机时间，看临界点的背景变化
           + 每次触发浇水动作时，除了直接观察界面效果，也会用charles抓包，看触发按钮后，接口请求是否反悔了正确的内容，主接口home返回的剩余水量值，所处的水量阶段是否正确
4. 优化点： +  一个是水滴雨小游戏玩的时候，游戏不流畅，有时会卡顿。提给开发，（css的优雅降级）
           +  另一个优化点是关于太阳道具每次点击会随机切换表情功能点的优化，当用charles抓包的时候发现，每次点击后接口会立马去同步状态，因为一共有五种表情这样对服务器的压力比较大，而且没与必要实时发送。后来优化成用户点击后两秒内若没有进行再次点击，才会去同步状态。
